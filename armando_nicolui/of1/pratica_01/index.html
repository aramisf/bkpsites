Entrada de Usuário no Sistema e Senhas

  1. Na tela inicial, clique duplamente em um dos nomes de máquina que se apresentam (dupond, dupont ou tournesol). Em seguida será apresentada a janela de login.

  2. Tecle seu nome de usuário (username ou login) e sua senha.

  3. Em seguida aparecerá em seu monitor um ambiente gráfico de janelas com alguns ícones à direita e à esquerda da tela. No conjunto de ícones no canto direito superior da tela, clique duplamente no ícone de um terminal de computador (geralmente o 2º ícone de cima para baixo).

    O surgimento de uma janela de shell, a exibição de uma eventual mensagem, e o prompt na janela de shell, indicam o sucesso da operação.

    OBS. 1: Outra forma de criar uma janela de shell é posicionar o cursor de mouse em uma região da tela sem ícones ou janelas (conhecida como root da área de trabalho) e clique o botão direito do mouse. No menu que se apresenta, clique a opção Xterminals. Aparecerá uma lista de opções de tipos de terminais de shell que se pode escolher. Escolha um deles com o mouse (posicione o cursor sobre uma das opções e clique).

    OBS. 2: prompt é um conjunto de caracteres (string) que indica que o sistema está pronto para aceitar comandos do usuário. Nos computadores do DInf, este prompt tem o seguinte formato:

    [user@máquina:/diretório_corrente]

  4. Para mudar a sua senha utilize o comando passwd. ATENÇÃO: você não conseguirá mudar a senha se não lembrar a senha antiga. Tenha também cuidado ao digitar a nova senha. Digite devagar e com firmeza para ter certeza do que está sendo digitado. Lembre-se também de pedir a pessoas próximas que se afastem para garantir que ninguém possa ver o que você está digitando.

  5. Execute o o comando passwd para mudar a sua senha para uma de sua preferência. Observe as recomendações da Seção 4.1 (Acesso ao Sistema), das Notas de Aula # 01. Procure mudar para uma senha que seja muito pessoal e fácil de você memorizar.

    Observe que o comando passwd não permite como senha palavras menores que 6 caracteres.

  6. Execute o comando ~ci066/bin/configuser, respondendo às perguntas que forem aparecendo na tela. Estas perguntas se referem à configuração de sua conta de e-mail no DInf.

  7. Termine suas atividades no sistema, efetuando o procedimento de logout. Para isto, posicione o cursor de mouse em uma região da tela sem ícones ou janelas (conhecida como raiz (root) da área de trabalho) e clique o botão direito do mouse.

    No menu que se apresenta, clique na última opção Exit. Na janela que se apresentar em seguida confirme a saída.

  8. Efetue login novamente, conforme os ítens 1 a 3.

  9. Execute o comando firefox. Este é o navegador WEB recomendado para uso no DInf. Observe na parte inferior de seu monitor o ícone associado ao navegador (aquele QUE NÃO TEM UMA UMA PEQUENA BARRA DE TÍTULO). Com o mouse, arraste este ícone para o conjunto de ícones no lado superior direito da tela.

    A partir de agora, nas próximas vezes que você entrar no sistema, basta clicar duplamente neste novo ícone para ativar o navegador.

  10. Acesse a URL http://www.inf.ufpr.br/nicolui/UFPR/Graduacao/CI066/1-2006/ e a acrescente na lista de Favoritos do navegador WEB (Bookmarks no firefox).

Comandos básicos Unix

Quando você efetua login, o UNIX/Linux assume um determinado diretório como sendo o diretório de trabalho inicial.

Este diretório inicial é conhecido dentro do jargão UNIX como diretório de login, diretório home, ou simplesmente home.

ATENÇÃO: Daqui para frente, nos textos produzidos para esta disciplina, este diretório será referenciado como home, $HOME ou ~.

  1. Verificando o diretório de login. Execute o comando pwd. Ele mostra ao usuário o diretório corrente. Imediatamente após o login, ele mostra qual é seu diretório de login. Procure memorizar o nome deste diretório. Ele é muito importante.

  2. Monitorando sua quota. Todo usuário possui uma quota, isto é, um espaço máximo em disco que este usuário pode ocupar com seus arquivos. O comando para verificar este espaço utilizado e seus limites é quota.

    Execute este comando com a opção -s e verifique o resultado que aparece na tela.

    NOTA: No resultado apresentado pelo comando quota, a coluna blocks indica o total de espaço em disco ocupado pelos seus arquivos. A coluna quota indica o limite permitido. A coluna limit indica o limite de tolerância em sua quota. Enquanto o valor da coluna blocks estiver entre os valores das colunas quota e limit, a coluna grace mostra uma mensagem com o prazo que você tem para limpar sua área e baixar o espaço ocupado por seu arquivos abaixo de sua quota. Durante este período de grace, o sistema ainda deixa você criar arquivos.

    Esgotado o prazo, o sistema dará sempre a mensagem de disk full ou Disk quota exceeded sempre que você tentar criar um arquivo (via cp, emacs ou qualquer outra forma de criação de arquivos).

  3. Verificando usuários no sistema. Execute o comando who para saber quem está compartilhando a estação com você.

  4. Use o comando whoami para identificar o seu prórpio username.

  5. Exibindo o conteúdo de um diretório. Execute as variantes do comando ls abaixo. Este comando mostra o conteúdo de um diretório.

    ci066@dupond:~$ ls
    ci066@dupond:~$ ls -a
    ci066@dupond:~$ ls -l

    Você consegue perceber a diferença entre os resultados da execução de cada linha de comando acima?

    NESTE PONTO CHAME O PROFESSOR PARA ESCLARECIMENTOS SOBRE O COMANDO ls

Editor EMACS

Tenha sempre à mão o conjunto de operações básicas do Emacs, disponíveis na Seção 4.3 (Editor de Texto - Emacs), das Notas de Aula # 01.

  1. Execute o comandos ls de forma a exibir na tela a lista dos arquivos em seu diretório de login, mostrando tamanho e data de criação dos arquivos.

  2. Execute o comando emacs.

    Observe na parte inferior de seu monitor o ícone associado ao emacs (aquele QUE NÃO TEM UMA UMA PEQUENA BARRA DE TÍTULO). Com o mouse, arraste este ícone para o conjunto de ícones no lado superior direito da tela.

    A partir de agora, nas próximas vezes que você entrar no sistema, basta clicar duplamente neste ícone para ativar o editor Emacs.

  3. Após o surgimento da janela do editor, digite um texto qualquer que tenha pelo menos 3 parágrafos, cada parágrafo com pelo menos 3 linhas.

  4. Salve o texto editado no arquivo de nome Primeira-Aula.txt, sem terminar a execução do editor. Use o menu existente na parte superior da janela do editor (Opção FILE  SAVE BUFFER AS)

  5. Repita o item 1 e observe a presença do arquivo editado e o seu tamanho.

  6. Treinamento com Editor EMACS Esta parte oferece alguns pequenos exercícios para que você exercite os comandos e operações do Emacs.

    Copie para sua área (diretório de login) o arquivo /home/especial/ci066/Arquivos/da_varanda.txt, que contém um singelo poema: 

                 cd
             cp /home/especial/ci066/Arquivos/da_varanda.txt .

  7. Edite o arquivo com o emacs e execute as operações indicadas abaixo. Para cada operação, anote a sequência de comandos emacs usada.

    ATENÇÃO: SEMPRE observe as mensagens da região de mensagens na janela do EMACS.

    Nas operações que envolvem posisionar o cursor de texto em algum lugar, não vale usar as setas de direção ou o mouse para posisionar o cursor;

    1. Substitua em todo o arquivo a seqüência ,cao por cao;
    2. Substitua em todo o arquivo a seqüência ao por ao;
    3. Posicione o cursor de texto na linha 19 e transforme todas as letras da palavra esguio em maiúsculas;
    4. Vá para o início do arquivo (primeira letra da primeira linha);
    5. Vá para o final da segunda estrofe;
    6. No verso com a palavra olhos, acrescente cabisbundo e sorumbatico, ao final da linha;
    7. Quantas vezes ocorre a palavra pinheiro?
    8. USANDO o mouse, marque toda a terceira estrofe e TRANSFIRA-A para a posição APÓS a sexta estrofe;
    9. Abra um novo arquivo chamado mais_varandas.txt em um novo buffer do emacs, mantendo o buffer do texto da_varanda.txt visível. DICA: Use Ctrl-x 4 f;
    10. Copie uma das estrofes do poema da_varanda.txt para mais_varandas.txt;
    11. Termine o emacs, salvando APENAS as alterações em mais_varandas.txt. NÃO SALVE as alterações feitas no arquivo da_varanda.txt.

Impressão de arquivos

Em UNIX, o comando para enviar o conteúdo de um arquivo para a impressora é lpr. Nos Laboratórios do DInf, esta impressora se chama salsicha e se localiza no Laboratório I.

Usa-se lpr apenas para arquivos texto/ASCII (do tipo gerado pelo editor emacs). Para imprimir páginas html, arquivos Word/OpenOffice e outros tipo de arquivos gerados por programas específicos, estes programas é que devem ser usados para efetuar a impressão.

OBS.: Em algumas distribuições de UNIX/Linux, o comando para impressão é lp.
